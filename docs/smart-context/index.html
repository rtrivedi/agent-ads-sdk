Smart Context Features

New in v0.15.1

Smart Context features provide 2-3x better CTR through automatic intent detection, interest extraction, and session tracking. Zero configuration required!

Overview​

Smart Context automatically understands your users' intent and interests, dramatically improving ad relevance without any extra work from you. Just pass conversation history and the SDK does the rest.

Performance Impact​

FeatureCTR ImprovementRevenue Impact
Intent Detection+35%+42%
Interest Matching+28%+31%
Session Tracking+15%+18%
Combined+65%+78%

How It Works​

Automatic Intent Detection​

The SDK automatically detects where users are in their journey:

const ad = await client.decideFromContext({
  userMessage: "Compare Pietra vs Shopify for dropshipping",
  conversationHistory: [
    "I want to start selling products online",
    "What platform should I use?"
  ]
});

// SDK automatically detects:
// ✓ intent_stage: 'comparison' (from "Compare X vs Y")
// ✓ interests: ['business', 'shopping', 'technology']
// ✓ purchase_intent: true (action-oriented language)

Intent Stages​

The SDK recognizes three intent stages:

1. Research Stage​

Users are learning and exploring:

// Detected from phrases like:
"what is...", "how does...", "explain...", "tell me about..."

// Example:
userMessage: "What is dropshipping and how does it work?"
// → intent_stage: 'research'

2. Comparison Stage​

Users are evaluating options:

// Detected from phrases like:
"vs", "versus", "compare", "difference", "which is better"

// Example:
userMessage: "Shopify vs WooCommerce for small business"
// → intent_stage: 'comparison'

3. Ready to Buy Stage​

Users are ready to take action:

// Detected from phrases like:
"buy now", "sign up", "get started", "pricing", "how much"

// Example:
userMessage: "How do I sign up for Shopify?"
// → intent_stage: 'ready_to_buy'

Interest Extraction​

The SDK automatically extracts user interests from conversations:

const ad = await client.decideFromContext({
  userMessage: "Best credit card for travel rewards",
  conversationHistory: [
    "Planning a trip to Europe",
    "Looking for flight deals"
  ]
});

// Automatically extracted interests:
// → ['travel', 'finance', 'shopping']

Supported Interest Categories​

The SDK recognizes 20+ interest categories:

  • Business: startup, entrepreneur, marketing, sales
  • Finance: money, invest, credit, loan, mortgage
  • Travel: vacation, flight, hotel, destination
  • Technology: software, app, AI, device
  • Health: medical, fitness, diet, wellness
  • Education: course, degree, certification
  • Shopping: store, discount, deal, sale
  • Home: real estate, furniture, renovation
  • And more...

Session Tracking​

Track multi-turn conversations for better context:

// First message in conversation
const ad1 = await client.decideFromContext({
  userMessage: "I need insurance",
  session_context: {
    session_id: 'sess_abc123',
    message_count: 1
  }
});

// Follow-up message
const ad2 = await client.decideFromContext({
  userMessage: "What about for a new driver?",
  conversationHistory: ["I need insurance"],
  session_context: {
    session_id: 'sess_abc123',  // Same session
    message_count: 2
  }
});

// SDK understands this is about car insurance for new drivers

Advanced Usage​

Explicit Context (Optional)​

While Smart Context works automatically, you can provide explicit hints for even better results:

const ad = await client.decideFromContext({
  userMessage: "What's the best option for me?",

  // Provide explicit user context
  user_context: {
    interests: ['wedding', 'photography'],
    recent_topics: ['wedding venues', 'photographers'],
    purchase_intent: true
  },

  // Provide session context
  session_context: {
    session_id: 'sess_wedding_planning',
    message_count: 5,
    intent_stage: 'ready_to_buy'
  }
});

Context Boost Mechanics​

The SDK boosts ad scores based on context signals:

  • Intent stage match: +20% score boost
  • Interest match: +15% per matching interest
  • High purchase intent: +10% score boost
  • Session continuity: +5% score boost
  • Maximum boost: Capped at 50% to maintain auction integrity

Best Practices​

1. Always Include Conversation History​

// ✅ Good - includes context
const ad = await client.decideFromContext({
  userMessage: "What about pricing?",
  conversationHistory: [
    "I need project management software",
    "Something for a small team"
  ]
});

// ❌ Bad - no context
const ad = await client.decideFromContext({
  userMessage: "What about pricing?"  // Pricing for what?
});

2. Use Session IDs for Multi-Turn Chats​

// Generate session ID once per conversation
const sessionId = `sess_${Date.now()}_${Math.random().toString(36).substring(7)}`;

// Use throughout the conversation
async function handleMessage(message: string, history: string[]) {
  return await client.decideFromContext({
    userMessage: message,
    conversationHistory: history,
    session_context: {
      session_id: sessionId,
      message_count: history.length + 1
    }
  });
}

3. Let Auto-Detection Work​

// ✅ Good - let SDK detect intent
const ad = await client.decideFromContext({
  userMessage: "Compare Node.js vs Python for web development",
  conversationHistory: history
});

// ❌ Avoid - unnecessary manual override
const ad = await client.decideFromContext({
  userMessage: "Compare Node.js vs Python",
  user_context: {
    interests: ['programming', 'web', 'development'],  // SDK detects this
  },
  session_context: {
    intent_stage: 'comparison'  // SDK detects this from "Compare"
  }
});

Real-World Examples​

Chatbot Integration​

class ChatbotWithAds {
  private client: AttentionMarketClient;
  private sessionId: string;
  private history: string[] = [];

  constructor() {
    this.client = new AttentionMarketClient({
      apiKey: process.env.AM_API_KEY!,
      agentId: process.env.AM_AGENT_ID!
    });
    this.sessionId = this.generateSessionId();
  }

  async handleMessage(userMessage: string) {
    // Get AI response
    const aiResponse = await this.generateResponse(userMessage);

    // Get contextual ad with Smart Context
    const ad = await this.client.decideFromContext({
      userMessage,
      conversationHistory: this.history.slice(-5), // Last 5 messages
      session_context: {
        session_id: this.sessionId,
        message_count: this.history.length + 1
      }
    });

    // Update history
    this.history.push(userMessage);

    return {
      response: aiResponse,
      sponsored: ad ? {
        title: ad.creative.title,
        body: ad.creative.body,
        url: ad.click_url
      } : null
    };
  }

  private generateSessionId(): string {
    return `sess_${Date.now()}_${Math.random().toString(36).substring(7)}`;
  }
}

Discord Bot with Context​

import { Client as DiscordClient } from 'discord.js';
import { AttentionMarketClient } from '@the_ro_show/agent-ads-sdk';

const amClient = new AttentionMarketClient({
  apiKey: process.env.AM_API_KEY!,
  agentId: process.env.AM_AGENT_ID!
});

// Track conversation history per channel
const channelHistory = new Map<string, string[]>();

bot.on('messageCreate', async (message) => {
  if (message.author.bot) return;

  const channelId = message.channel.id;
  const history = channelHistory.get(channelId) || [];

  // Get ad with channel context
  const ad = await amClient.decideFromContext({
    userMessage: message.content,
    conversationHistory: history.slice(-10), // Last 10 messages
    session_context: {
      session_id: `discord_${channelId}`,
      message_count: history.length + 1
    }
  });

  // Update history
  history.push(message.content);
  if (history.length > 50) history.shift(); // Keep last 50
  channelHistory.set(channelId, history);

  // Send response with ad if relevant
  if (ad && ad.relevance_score > 0.7) {
    await message.reply({
      embeds: [createAdEmbed(ad)]
    });
  }
});

FAQ​

Do I need to change my existing code?

No! Smart Context works automatically. Your existing code will benefit immediately after upgrading to v0.15.1.

Will this increase latency?

No. Intent detection and interest extraction happen in-memory with negligible overhead (less than 5ms).

Is conversation history sent to advertisers?

No. We only extract intent signals and interests. Raw conversation data never leaves our servers and is never shared with advertisers.

What if intent detection is wrong?

The system is designed to fail gracefully. Even if intent is misclassified, the semantic matching still ensures relevant ads based on the actual message content.

Can I disable Smart Context?

While not recommended (you'll lose the CTR benefits), you can disable it by not passing conversationHistory or session_context.
