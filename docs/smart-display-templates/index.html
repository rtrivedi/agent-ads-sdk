Smart Display Templates

Intelligent ad placement patterns and ready-to-use templates for optimal user experience and revenue.

Placement Decision Engine​

When to Show Ads​

Use this decision tree to determine the optimal moment for ad placement:

class SmartPlacementEngine {
  shouldShowAd(context: ConversationContext): PlacementDecision {
    // Never show ads during:
    if (context.isOnboarding) return { show: false };
    if (context.hasRecentError) return { show: false };
    if (context.messageCount < 3) return { show: false };
    if (context.lastAdShownAt && context.messageCount - context.lastAdShownAt < 5) {
      return { show: false };
    }

    // Optimal placement scenarios
    if (context.hasCommercialIntent) {
      return {
        show: true,
        placement: 'inline',
        priority: 'high'
      };
    }

    if (context.isTaskComplete) {
      return {
        show: true,
        placement: 'after_response',
        priority: 'medium'
      };
    }

    if (context.conversationPaused > 2000) { // 2 second pause
      return {
        show: true,
        placement: 'gentle_suggestion',
        priority: 'low'
      };
    }

    return { show: false };
  }
}

Display Templates by Context​

1. Inline Contextual (During Conversation)​

When to use: User explicitly mentions a need or problem

// Detect commercial intent in message
if (userMessage.includes('need') || userMessage.includes('looking for')) {
  const ad = await client.decideFromContext({
    userMessage,
    conversationHistory,
    placement: 'contextual_promotion'
  });

  // Display inline with conversation flow
  return {
    type: 'inline_suggestion',

        <div class="ai-response">${aiResponse}</div>

            ${ad.creative.teaser ||

            ${ad.creative.cta} →
          </a>
        </div>
      </div>

  };
}

2. Post-Response Suggestion (After AI Answer)​

When to use: After providing value, suggest related services

// After AI completes its response
async function showPostResponseAd(conversation: Conversation) {
  // Wait for natural pause
  await sleep(1500);

  const ad = await client.decideFromContext({
    userMessage: conversation.lastUserMessage,
    conversationHistory: conversation.messages,
    placement: 'sponsored_suggestion'
  });

  return {
    type: 'follow_up_suggestion',
    template: `
      <div class="follow-up-ad">
        <div class="divider">• • •</div>
        <div class="sponsored-follow-up">
          <img src="/sponsor-icon.svg" class="sponsor-badge" />
          <div class="follow-up-content">
            <p class="follow-up-intro">Related suggestion:</p>
            <h4>${ad.creative.title}</h4>
            <p>${ad.creative.body}</p>
            <button onclick="window.open('${ad.click_url}')">
              ${ad.creative.cta}
            </button>
          </div>
          <span class="disclosure">Sponsored by ${ad.disclosure.sponsor_name}</span>
        </div>
      </div>
    `
  };
}

3. Natural Break Placement​

When to use: During conversation pauses or topic transitions

class NaturalBreakDetector {
  detectBreak(messages: Message[]): boolean {
    const lastMessage = messages[messages.length - 1];
    const timeSinceLastMessage = Date.now() - lastMessage.timestamp;

    // Natural pause detected
    if (timeSinceLastMessage > 3000) return true;

    // Topic transition detected
    if (this.isTopicChange(messages)) return true;

    // Task completion detected
    if (lastMessage.text.match(/done|completed|finished|thanks/i)) return true;

    return false;
  }
}

4. Conversational Teaser Pattern​

When to use: For recommendation ads with high relevance

// Recommendation ad with teaser approach
async function showTeaserAd(ad: RecommendationAd) {
  // Step 1: Natural teaser question
  await displayMessage({
    type: 'ai_followup',
    content: ad.creative.teaser ||
             "By the way, I found something that might help with this.",
    showTypingIndicator: true
  });

  // Step 2: Wait for engagement signal
  const engaged = await waitForUserEngagement(2000);

  if (engaged) {
    // Step 3: Show full promotion
    await displayMessage({
      type: 'sponsored_recommendation',
      content: `
        <div class="recommendation-card">
          <span class="promo-badge">Special Offer</span>
          <h4>${ad.creative.title}</h4>
          <p>${ad.creative.body}</p>
          ${ad.creative.promo_code ?
            `<div class="promo-code">Code: ${ad.creative.promo_code}</div>` : ''}
          <a href="${ad.click_url}" class="cta-button">
            ${ad.creative.cta}
          </a>
          <small>Sponsored</small>
        </div>
      `
    });
  }
}

Timing Algorithms​

Progressive Disclosure​

Show ads with increasing frequency as engagement grows:

class ProgressiveDisclosure {
  calculateAdFrequency(session: Session): number {
    const baseInterval = 10; // messages
    const engagementMultiplier = Math.min(session.engagementScore / 100, 1);
    const sessionLengthBonus = Math.min(session.messageCount / 50, 0.5);

    // More engaged users see slightly more relevant ads
    const adjustedInterval = baseInterval * (1 - engagementMultiplier * 0.3);

    // Longer sessions get better spacing
    return Math.ceil(adjustedInterval + sessionLengthBonus * 5);
  }
}

Smart Batching​

Prevent ad fatigue by batching related suggestions:

class SmartBatcher {
  async batchRelatedAds(context: Context): Promise<Ad[]> {
    // Get multiple relevant ads at once
    const ads = await client.getMultipleAds({
      userMessage: context.userMessage,
      maxAds: 3,
      minQualityScore: 0.8
    });

    // Show as carousel or grouped suggestions
    return {
      type: 'grouped_suggestions',
      title: 'Related options you might like:',
      ads: ads.slice(0, 2), // Limit to 2 for better UX
      showAllAtOnce: false,
      delayBetween: 1000
    };
  }
}

Claude Code Integration​

Ready-to-use patterns for AI assistants:

// For Claude Code and similar AI assistants
class AIAssistantAdIntegration {
  async handleUserQuery(query: string): Promise<Response> {
    // Step 1: Process user query normally
    const aiResponse = await generateResponse(query);

    // Step 2: Determine if ad is appropriate
    const adContext = {
      hasCommercialIntent: this.detectCommercialIntent(query),
      responseCompleted: true,
      conversationLength: this.messageHistory.length
    };

    if (!this.shouldShowAd(adContext)) {
      return { content: aiResponse };
    }

    // Step 3: Get contextual ad
    const ad = await attentionMarket.decideFromContext({
      userMessage: query,
      conversationHistory: this.messageHistory
    });

    if (!ad) {
      return { content: aiResponse };
    }

    // Step 4: Compose response with natural ad placement
    return {
      content: aiResponse,
      followUp: {
        delay: 1500,
        content: this.formatSponsoredSuggestion(ad)
      }
    };
  }

  formatSponsoredSuggestion(ad: Ad): string {
    // Natural language format for AI responses
    return `\n\n---\n*Related suggestion:* ${ad.creative.title}\n\n` +
           `${ad.creative.body}\n\n` +
           `[${ad.creative.cta}](${ad.click_url}) ` +
           `*(Sponsored by ${ad.disclosure.sponsor_name})*`;
  }
}

A/B Testing Templates​

Test different display patterns:

enum DisplayVariant {
  SUBTLE = 'subtle',        // Minimal, text-only
  CARD = 'card',           // Rich card with borders
  INLINE = 'inline',       // Flows with conversation
  MODAL = 'modal'          // Attention-grabbing overlay
}

class ABTestingFramework {
  selectVariant(user: User): DisplayVariant {
    // Deterministic variant selection
    const hash = this.hashUserId(user.id);
    const variants = Object.values(DisplayVariant);
    return variants[hash % variants.length];
  }

  trackPerformance(variant: DisplayVariant, event: ClickEvent): void {
    analytics.track('ad_variant_performance', {
      variant,
      clicked: event.clicked,
      timeToClick: event.timeToClick,
      relevanceScore: event.ad.relevanceScore
    });
  }
}

Performance Metrics​

Track display effectiveness:

interface DisplayMetrics {
  viewability: number;      // % of ad actually seen
  timeInView: number;       // Seconds ad was visible
  engagementRate: number;   // % who interacted
  clickThrough: number;     // % who clicked
  dismissRate: number;      // % who explicitly closed
}

class MetricsCollector {
  track(ad: Ad, display: DisplayTemplate): void {
    // Use Intersection Observer for viewability
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            this.startViewTimer(ad.id);
          } else {
            this.stopViewTimer(ad.id);
          }
        });
      },
      { threshold: 0.5 } // 50% visible
    );

    observer.observe(display.element);
  }
}

Best Practices​

Do's​

  • ✅ Wait for natural breaks in conversation
  • ✅ Ensure ads add value to the user experience
  • ✅ Test different placements with your audience
  • ✅ Respect user preferences and frequency caps
  • ✅ Make sponsored content clearly labeled

Don'ts​

  • ❌ Show ads during error states
  • ❌ Interrupt user input or active typing
  • ❌ Display multiple ads simultaneously
  • ❌ Use deceptive patterns or hidden ads
  • ❌ Show ads before providing value
